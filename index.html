Comparison Tables

'Responsive tables' as a flexible reusable component

  1. Being responsive must be optional
  2. Vertical scrollbars is a valid option
  3. Collapse by rows if content is arranged horizontally
  4. Collapse by columns if content is arranged vertically
  5. Collapse to tabs or accordions

Limitations of standard
<table> markup

  • Not as accessible as you might think
    • No way to make a screen reader read vertically down columns
    • <thead>, <th> and scope don't seem to do anything
  • Collapsing responsively by 'columns' is harder than ruling 7 kingdoms

Example 'Best Practice' table

How not to build a responsive table

  1. Generate a second responsive table and toggle on breakpoint
    CONS: Code duplication/bloat. Will duplicate unique IDs inside a table
  2. Rearrange the table with JS on breakpoint
    CONS: Requires JS & screen events, Must re-initialize contents on change
  3. Switch from <table> to display:flex
    CONS: Simply not possible to do vertical/horizontal & collapse row/column with the same markup
